Note de synthèse première année



Note de Synthèse n°1 : Intégration de MQTT dans une Interface Web

Cette première note de synthèse réalisée durant mon expérience chez Orange porte sur l'intégration du protocole MQTT (Message Queuing Telemetry Transport) dans une interface web. L'objectif était de permettre la gestion et le test des différentes requêtes envoyées au système directement depuis un navigateur.

Objectif du projet

L'objectif principal était de développer une interface web permettant d'interagir avec un broker MQTT afin de simplifier les phases de test et de validation des communications entre les différents services.

  • Connexion à un broker MQTT
  • Publication de messages sur différents topics
  • Souscription aux topics pour recevoir les réponses
  • Visualisation des messages en temps réel

Technologies utilisées

HTML

Structure de l'interface web

CSS

Mise en forme et ergonomie

JavaScript

Gestion des interactions

MQTT.js

Communication avec le broker

Résultats obtenus

L'intégration de MQTT dans une interface web permet de tester rapidement les communications entre les services et d'observer en temps réel les échanges de messages. Cette solution facilite le débogage et la validation des requêtes envoyées au système.

Note de synthèse deuxième année



Note de Synthèse n°2 : Tests unitaires avec Robot Framework

Cette seconde note de synthèse, réalisée durant mon stage chez Orange, porte sur la mise en place de tests unitaires automatisés à l’aide de Robot Framework. Ce projet s’inscrivait dans un programme plus large visant à améliorer un logiciel interne dédié à la gestion des droits utilisateurs.

Objectif du projet

L'objectif principal était de développer une suite de tests automatisés permettant de vérifier le bon fonctionnement d'une API développée en Go. Cette API faisait partie d’un système de gestion des droits utilisateurs utilisé pour contrôler l’accès aux différentes ressources du logiciel.

  • Automatiser les tests de requêtes vers l’API
  • Vérifier les réponses et les comportements attendus
  • Détecter rapidement les régressions lors des mises à jour
  • Faciliter la validation des fonctionnalités du système

Technologies utilisées

Python

Langage utilisé pour l’écriture des tests

Robot Framework

Automatisation des scénarios de test

API REST

Interface testée par les scripts

Go

Langage de l’API testée

Rôle dans le projet

Mon travail consistait à concevoir et implémenter des scénarios de tests permettant de vérifier les différentes fonctionnalités de l’API. Bien que l’API soit développée en Go, mon intervention portait uniquement sur la création et l’exécution des tests en utilisant Robot Framework et Python.

Résultats obtenus

La mise en place de ces tests automatisés a permis de renforcer la fiabilité du système en vérifiant automatiquement le comportement de l’API lors des évolutions du logiciel. Ce travail contribue à améliorer la qualité du projet global dédié à la gestion des droits utilisateurs.
